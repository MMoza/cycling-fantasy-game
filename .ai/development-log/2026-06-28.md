# 2026-06-28

## Trabajo realizado

### Capa de Application (Use Cases + DTOs)

Creada la capa `app/Application/` siguiendo la arquitectura DDD:

- **36 Use Cases** organizados por dominio: Dashboard, League, Stage, Prediction, Classification, y Admin (Competition, Edition, Stage, FinalClassification, CompetitionSetup, Team, Rider, User)
- **15 DTOs** inmutables para tipar la entrada/salida de cada Use Case
- **ApplicationException** personalizada para errores de dominio, evita que `abort()` sea capturada por controladores

Cada Use Case encapsula una operación de negocio completa, sin dependencia directa de Eloquent en los controllers.

### Refactorización de controllers

Los 16 controllers públicos y de admin ahora delegan toda la lógica a Use Cases mediante inyección de dependencias por constructor:

- `DashboardController` -> `ShowDashboardUseCase`
- `LeagueController` -> 5 Use Cases
- `PredictionController` -> 3 Use Cases
- `StageController` -> 2 Use Cases
- `ClassificationController` -> 1 Use Case
- Admin controllers -> 28 Use Cases

**Cero lógica de negocio** en controllers. Solo validación de input, llamada al Use Case, y renderizado Inertia.
