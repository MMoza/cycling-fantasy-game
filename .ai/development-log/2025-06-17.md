# 2025-06-17 — Fase 1 Paso 1: Instalar Laravel Breeze con Inertia + React + TypeScript

## Objetivos
- Instalar Laravel Breeze con stack React + TypeScript
- Configurar autenticación con Inertia
- Verificar que el build y las rutas funcionan

## Trabajo realizado
- Instalado `laravel/breeze` v2.4.2 via Composer
- Ejecutado `php artisan breeze:install react --typescript`
- Instaladas dependencias npm con `--legacy-peer-deps` (conflicto vite@8 vs @vitejs/plugin-react@4)
- Creado manualmente `resources/js/bootstrap.ts` (no fue generado por Breeze)
- Verificado build con `npm run build` → éxito
- Verificado rutas de auth con `php artisan route:list --path=login` → OK
- Verificado migraciones con `php artisan migrate:fresh` → OK

## Decisiones tomadas
- Usar `--legacy-peer-deps` para resolver conflicto de dependencias de Vite 8
- Crear `bootstrap.ts` manualmente con configuración estándar de axios

## Problemas encontrados
- Breeze no generó el archivo `bootstrap.ts` pero `app.tsx` lo importa
- Conflicto de dependencias npm: vite@8 requiere @vitejs/plugin-react que solo soporta hasta vite@7

## Solución aplicada
- Creado `resources/js/bootstrap.ts` manualmente con configuración de axios
- Usado `npm install --legacy-peer-deps` para resolver el conflicto

# 2025-06-17 — Fase 1 Paso 4: Inicializar shadcn/ui

## Objetivos
- Inicializar shadcn/ui con configuración neutral estilo Vercel
- Añadir CSS variables para light/dark theme
- Crear utilidades base (cn helper)
- Añadir primer componente (Button)

## Trabajo realizado
- Ejecutado `npx shadcn@latest init -d` con:
  - Style: base-nova
  - Base color: neutral
  - Icon library: lucide
  - CSS variables: true
- Creado `components.json` con aliases `@/`
- Creado `resources/js/lib/utils.ts` con función `cn()` (clsx + tailwind-merge)
- Añadidas CSS variables para light/dark theme en `app.css`
- Instaladas dependencias: `@base-ui/react`, `clsx`, `tailwind-merge`, `class-variance-authority`, `lucide-react`
- Añadido Button component desde shadcn registry
- Fix CSS para compatibilidad con Tailwind v3 (no usar @apply con clases custom)
- Build verificado, 25 tests Pest + 1 test Vitest pasando

## Problemas encontrados
- shadcn init falla al instalar dependencias por conflicto vite@8 vs @vitejs/plugin-react@4
- Button component usa `@base-ui/react` que no se instaló automáticamente
- CSS `@apply border-border` no funciona con Tailwind v3

## Solución aplicada
- Instaladas dependencias manualmente con `--legacy-peer-deps`
- Añadido `@base-ui/react` manualmente
- Reemplazado `@apply border-border` por `border-color: hsl(var(--border))`
