# .ai/ --- Documentación Viva del Proyecto

Este directorio contiene la documentación técnica y funcional del proyecto PseudoFantasy Cycling.
Está diseñada para ser útil tanto para humanos como para agentes de IA.

## Estructura

| Directorio | Propósito |
|---|---|
| `development-log/` | Registro diario de trabajo realizado y decisiones |
| `adr/` | Architecture Decision Records (decisiones arquitectónicas) |
| `domain/` | Documentación funcional del dominio (reglas, invariantes) |
| `incidents/` | Registro de bugs relevantes y su resolución |
| `runbooks/` | Guías operativas (despliegue, backups, etc.) |
| `prompts/` | Prompts reutilizables para agentes IA |

## Principios

1. La documentación es código: versionada, revisada, mantenida.
2. Toda decisión importante queda registrada en un ADR.
3. El conocimiento no reside en la cabeza de nadie.
4. Documentación útil para humanos Y agentes IA.
5. Información estructurada y fácilmente consultable.

## Cómo usar

- **Antes de empezar una tarea**: Revisa ADRs y documentación del dominio relevante.
- **Tras completar una tarea**: Actualiza el development log del día.
- **Al tomar una decisión arquitectónica**: Crea un ADR.
- **Al encontrar un bug relevante**: Registra un incidente.
